## 1. このスキルについて

このスキルは、ユーザーとの議論内容を構造化されたMarkdownノートとして自動記録する。以下のタイミングで発動する：

- **明示的な指示**: 「まとめて」「整理して」「記録して」などの指示を受けた時
- **Proactive発動**: 以下の条件を満たした場合、自動的にノート作成を提案する
  - 議論が5往復以上続き、一定の結論やインサイトが得られた
  - アイデアや設計方針が具体化し、記録する価値がある内容になった
  - 話題が自然に一段落し、次のトピックに移りそうな雰囲気

## 2. ファイルの生成と命名規則

以下のルールでファイルを作成する：

- **保存先**: `notes/` ディレクトリ（存在しない場合は作成）
- **ファイル名**: `YYYY-MM-DD_トピック名.md`
  - 例: `notes/2026-03-04_アプリのDB設計.md`
- **既存ファイルへのリンク**: 議論の過程で参照した既存のMDファイルがあれば、リポジトリルートからの相対パスでリンクを記載する
  - 例: `[関連ノート](./2026-03-01_認証設計.md)`

## 3. Markdownのフォーマット

生成するファイルは、標準的なMarkdown記法を用い、後から見返しやすいように以下の構成で出力する：

```md
# [トピック名]

- **Date:** YYYY-MM-DD
- **Keywords:** [関連キーワード1], [関連キーワード2]

## 議論の目的と背景
（ここに簡潔な背景を記載）

## 決定事項・重要なインサイト
（箇条書きで分かりやすく整理）

## 次のアクション・未解決の課題
（もしあれば記載）

## 関連ノート
（参照した既存ノートがあれば相対パスでリンク）
```
